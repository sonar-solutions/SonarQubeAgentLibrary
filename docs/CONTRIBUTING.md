## How to Contribute

### Reporting Issues
If you find a bug or have a suggestion:
1. Check if the issue already exists
2. Open a new issue with a clear title and description
3. Include steps to reproduce (for bugs)
4. Suggest expected behavior

### Submitting New Agents

We welcome new agents! Follow these guidelines:

#### 1. Agent Requirements
- **Clear purpose**: Agent should solve a specific problem
- **Well-defined persona**: Clear role and expertise
- **Actionable prompts**: Include 4-6 suggested prompts
- **Official references**: Link to authoritative documentation
- **Security-conscious**: Emphasize secure practices

#### 2. File Structure
- Place agent in `.github/copilot-agents/`
- Use kebab-case naming: `agent-name.agent.md`
- Follow the template structure (see existing agents)

#### 3. Testing Your Agent
Before submitting:
- Test all suggested prompts
- Verify links are valid and current
- Ensure persona is consistent
- Check for typos and grammar

#### 4. Pull Request Process
1. Fork the repository
2. Create a feature branch: `git checkout -b feature/new-agent-name`
3. Add your agent file
4. Update README.md to list your agent
5. Commit with clear message: `git commit -m "Add [Agent Name] for [purpose]"`
6. Push and create a pull request
7. Respond to review feedback

### Improving Existing Agents

To improve an agent:
1. Open an issue describing the improvement
2. Get feedback from maintainers
3. Make the changes
4. Submit a pull request

### Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Help others learn and grow
- Follow the project's style guidelines

## Questions?

Open a discussion or reach out to the maintainers!
